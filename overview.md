# Rhygym（リジム）— アプリ概要書

> 楽譜を読み、画面タップでリズムを叩く。読譜力を鍛えるWebリズム学習アプリ。

**アプリ名**：Rhygym（リジム）。rhythm（リズム）と gym（ジム＝鍛える場）を掛け合わせた造語。「リズムを鍛えるジム」というコンセプトを名前に込めている。表記は1語の `Rhygym` を正式名称とし、ロゴでは `RhyGym` のように rhythm と gym の切れ目を見せる表現も可とする。

本書はアプリの全体像をまとめた概要書です。詳細な内部設計（モジュール構成・データ構造・判定アルゴリズムなど）は本書には含めず、別途設計フェーズで扱います。

---

## 1. コンセプト

楽譜上のリズム譜を読み取り、音符のタイミングに合わせて画面をタップする音ゲー型の学習アプリ。「楽譜からリズムを認識する力（読譜力）」を、ゲーム感覚で段階的に鍛えることを目的とする。

洗足オンラインスクールのリズム学習ソフトを参考にしているが、本アプリはスマートフォンのブラウザでの利用を主軸に据え、タッチ操作で完結する設計とする。

---

## 2. 目的と背景

ソルフェージュ（音楽の基礎訓練）の中でも、リズムの読譜は独学が可能な領域である。一方で、四分音符・八分音符・付点・連符などのリズム表記を「見た瞬間に正しい長さで認識する」感覚は、反復練習でしか身につきにくい。

本アプリは、その反復を退屈な作業ではなくゲーム体験に変えることを狙う。流れてくる楽譜を読み、正しいタイミングで叩くという操作を通じて、譜面と実際のリズムが自然に結びつくことを目指す。

---

## 3. ターゲットユーザー

- リズム譜の読譜を練習したい音楽初学者・学習者
- 楽器や音楽理論を学ぶ学生
- 隙間時間にスマホで気軽にトレーニングしたい人

専門知識がなくても、最初の級から始めれば無理なく取り組める難易度設計とする。

---

## 4. コア体験

1. 級（難易度）を選ぶ
2. カウントイン後、楽譜が流れ始める
3. 音符が判定ラインに重なる瞬間に画面をタップ
4. タイミングのズレに応じて判定（PERFECT / GOOD / MISS）が出る
5. 演奏終了後、スコア・ランク・正確さがリザルトとして表示される

「楽譜を読む」→「タイミングで叩く」→「結果で振り返る」というループを軸とする。

---

## 5. 主な機能

- **級別ステージ**：基本リズム（四分・二分音符）から、休符・八分音符・付点・十六分音符、そして全要素を組み合わせた総合ステージまで、段階的な難易度を用意する。
- **楽譜表示**：VexFlow を用いて、正しい楽譜表記でリズム譜を描画する。学習用途のため、表記の正確さを重視する。
- **タイミング判定**：タップと音符の時間差を計測し、複数段階で判定する。
- **メトロノーム / カウントイン**：拍を音で感じられるよう、クリック音と演奏開始前のカウントインを鳴らす。
- **スコアとランク**：判定内容に応じたスコア集計と、演奏全体の評価ランクを表示する。
- **プレイデータの保存**：各級のベストスコア・ランクなどをブラウザに保存し、次回以降も記録が残るようにする。

---

## 6. 画面構成

| 画面 | 役割 |
|------|------|
| タイトル | アプリの入口。コンセプト提示と開始導線。 |
| 級選択 | 各級の一覧。難易度・収録リズム・自己ベストを表示し、ここから開始する。 |
| ゲーム | 楽譜表示・タップ操作・判定・スコア表示を行うメイン画面。 |
| リザルト | 演奏結果（ランク・スコア・判定内訳）の表示と、再挑戦／級選択への導線。 |

---

## 7. 技術スタック

- **フロントエンド**：React
- **楽譜描画**：VexFlow
- **データ保存**：ブラウザの localStorage（端末ローカルに記録を保持）
- **公開**：GitHub Pages（静的サイトとしてホスティング）

React は画面遷移や各種状態（級選択・リザルトなど）の管理に用いる。楽譜描画は VexFlow を命令的に呼び出して行う。リズムゲームの中核となるタイミング処理・音声・アニメーションは、React の描画サイクルとは独立して動作させる方針とする。

### 補足：データ保存について

GitHub Pages は静的ホスティングであり、サーバー側の処理は持てない。そのため初版のデータ保存は localStorage に限定し、記録は利用者の端末内に保存される。複数端末での共有やオンラインランキングは初版のスコープ外とする（第9章参照）。

---

## 8. スコープ（初版）

### 初版で実現すること

- タイトル／級選択／ゲーム／リザルトの4画面
- 複数の級によるリズム譜ステージ（全レベル対応）
- VexFlow による楽譜表示
- タップ操作によるタイミング判定とスコアリング
- localStorage による自己ベストの保存
- スマートフォンのブラウザでの動作

### 初版では扱わないこと

- オンラインランキング、サーバーへのデータ保存
- ユーザーアカウント／ログイン
- 楽曲音源に合わせた演奏（初版はメトロノームベース）
- 譜面の自作・編集機能
- マイクによる実楽器の音の判定

---

## 9. 今後の拡張余地

- **サーバー記録 / ランキング**：Firebase などの外部サービスを併用し、オンラインランキングや複数端末でのデータ共有を実現する。
- **譜面の拡充**：級や課題曲を追加する。譜面データを編集しやすい形で持たせ、拡張を容易にする。
- **リズムの読み方表示**：「タン・タタ」などの音節表記を併記し、読譜の補助とする。
- **演奏モードの追加**：左右の手拍子を分けるレーン制、楽曲音源との同期演奏など。
- **譜面エディタ**：利用者が自分でリズム譜を作成できる機能。

---

## 10. 用語メモ

- **Rhygym（リジム）**：本アプリの名称。rhythm + gym の造語。
- **読譜**：楽譜を読み取り、音楽として理解すること。本アプリではリズム譜の読み取りを指す。
- **判定ライン**：流れてくる音符が重なった瞬間がタップのタイミングとなる、画面上の基準線。
- **カウントイン**：演奏開始前に拍を数える助走部分（例：4・3・2・1）。
